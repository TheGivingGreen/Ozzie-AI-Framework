# Documentation Standard

Documentation should help a capable person or agent act with confidence.

## Principles

- Start with purpose before procedure.
- Name the intended audience.
- Prefer direct instructions over vague recommendations.
- Keep examples generic unless a template explicitly asks for project-specific details.
- Use consistent terms across files.
- Update navigation when adding important documents.

## Required Shape

Every major document should include:

- Title
- Purpose
- Scope or audience
- Steps, standards, or decisions
- Ownership or update guidance when relevant

## Writing Style

Use plain language. Avoid filler, internal shorthand, and unexplained acronyms.

Agents should be able to copy instructions into a new project without needing to infer missing context.
